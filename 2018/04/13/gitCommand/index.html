Git 常用的命令行

Posted on 2018-04-13 Edited on 2021-02-03
Symbols count in article: 1k Reading time ≈ 1 mins.

简介

本文主要记录一下在工作中经常使用的git命令。

Git 初始化

git init 常用来初始化git 仓库

git 提交命令

git add 用来添加修改的变化

git add . 是用来添加所有的修改变化

git commit 是用来提交到本地仓库

git commit -a -m"描述信息"

git pull 远端的仓库

git pull 拉取远端的仓库

当在使用pull的时候会发生冲突，如果是iOS开发建议使用Xcode来解决冲突,(冲突主要是集中在<<<< 在这中间>>>)

git pull -p 强制更新本地大分支相关

git push 到远端的仓库

git push  

git push origin HEAD:refs/for/CarKeyUserDev

git tag

git tag -l 查看所有的tag

git tag -v +tag的标签内容

git checkout tag-name 根据tag的name拉取相应tag位置的代码

git tag -a 版本号 -m '版本描述'

git push -a tags 将所有的tag push到远端

git branch

git branch -a 查看所有的branch

git checkout branchName  切换分支

git branch -D branchName 删除相应的分支

git checkout -b branchName 新建新的分支

git cherry－pick

cherry-pick 命令“复制”一个提交节点并在当前分支做一次完全一样的新提交。

git rebase

衍合是合并命令的另一种选择。合并把两个父分支合并进行一次提交，提交历史不是线性的。衍合在当前分支上重演另一个分支的历史，提交历史是线性的。本质上，这是线性化的自动的 cherry-pick。

git reflog

git reflog 引用日志

find .git/objects -type f | xargs ls -lt | sed 30q 查看最近git add 的操作

git fsck –full

查找本地删除掉的commitid
每次的commit都会在git文件夹中，就算删除掉的

delete Branch & tag

删除分支    
$ git push origin --delete <branchName>
删除tag
git push origin --delete tag <tagname>
